# Markdown 文档元数据与免责规范

## 适用范围

- 本规则适用于**新增**或**实质性修订**的 Markdown（.md）文件，尤其是：
  - `docs/` 下 PRD、设计、架构、说明类文档；
  - `prd/` 下 PRD 与版本规划类文档；
  - 工程/产品类知识文档（如 `docs/engineering/`、`docs/api/` 等）。
- 代码内联注释、临时个人笔记可酌情简化，但建议仍带日期或免责。

## 必须项

### 1. 最新修订日期时间

- **必须**在文档中显式标注该文档的**最新修订日期时间**，**精确到秒**，即该文档创建或修改时的实际时间。
- 建议位置：文档开头（标题下）或文末「修订记录」。
- 格式示例（必须含时分秒）：
  - `**成文日期**：2026-02-15 20:10:00 UTC+8`
  - `**最后更新**：2026-02-15 20:10:00 UTC+8`
  - `**最后修订**：2026-02-15 20:10:00 UTC+8`

### 2. 免责说明

- **必须**在文档中显式标注**免责或时效性说明**，提示读者实现/产品可能已变化，需以当前系统为准。
- 建议位置：紧接在修订日期之后、正文之前（文首块内）。
- 表述示例（任选其一或等价表述）：
  - `**阅读时当前系统实现可能已发生变化，请以实际代码与产品行为为准，谨慎参考。**`
  - `**阅读时系统实现可能已经与本文档不一致，谨慎参考。**`
  - `**本文档为历史/快照记录，系统现状可能已改变，仅供参考。**`

## 推荐结构（文首示例）

```markdown
# 文档标题

**成文日期**：YYYY-MM-DD HH:mm:ss UTC+8
**最后修订**：YYYY-MM-DD HH:mm:ss UTC+8

本文档……（一两句说明文档目的与存放路径）。

**阅读时当前系统实现可能已发生变化，请以实际代码与产品行为为准，谨慎参考。**

---

## 一、正文
```

## 校验要点（Code Review）

- 新增或大改的 .md 是否包含「最新修订日期时间」（精确到秒）？
- 日期时间是否随本次修订更新？
- 是否包含「免责/谨慎参考」类说明？
